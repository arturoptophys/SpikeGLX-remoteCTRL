API reference¶

SpikeGLX-remoteCTRL¶

class spikeGLX_remote.spikeGLXremote_ctrl.SpikeGLX_Controller(main=None)[source]¶

This class uses the spikeGLX-api to control the SpikeGLX recorder. Needs to run on a windows computer due to only dlls being available. This application receives commands from the remote main TaskController via sockets, it can receive the session name for the recording, start/stop recording, start viewing, copy files to the data server, purge files.

Parameters:

  • main (GUI_utils.MainWindow) – reference to the main GUI

  • remote_thread_stop (threading.Event) – Event to stop the remote thread

  • remote_thread (threading.Thread) – thread to check for remote messages

  • files_copied (bool) – flag if files were copied

  • session_path (Path) – path to copy the files to

  • is_remote_ctr (bool) – flag if in remote control mode

  • rec_start_time (float) – time when recording started

  • hSglx (ctypes.c_void_p) – handle to the spikeglx api connection

  • is_recording (bool) – flag whether currently recording

  • is_viewing (bool) – flag whether currently viewing

  • session_id (str) – placeholder for the current session id

  • recording_file (Path) – placeholder for the recording file

  • log (logging.Logger) – logger object

  • socket_comm (SocketComm) – socket communication object

  • _save_path (Path) – path to save the recorded files

  • last_t_socket (float) – last time we checked for a message from the remote controller

  • check_interval (int) – check interval for messages

  • can_copy (bool) – flag if files can be copied
